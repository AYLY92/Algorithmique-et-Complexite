2.2 Gradient Descent¶

Dans cette partie, vous adapterez les paramtres de rgression linaire $ \theta $ notre ensemble de donnes en utilisant la descente en gradient.

2.2.1 Update Equations¶

L'objectif de la rgression linaire est de minimiser la fonction de cot

$$ J(\theta) = \frac{1}{2m} \sum_{i=1}^m \left( h_{\theta}(x^{(i)}) - y^{(i)}\right)^2$$

o la fonction d'hypothse $h_\theta(x)$ est donn par le modle linaire $$ h_\theta(x) = \theta^Tx = \theta_0 + \theta_1 x_1$$

Rappelez-vous que les paramtres de votre modle sont les valeurs $ \ theta_j $. Ceux-ci sont les valeurs que vous ajusterez pour minimiser le cot $ J (\ theta) $. Une faon de le faire est de utiliser l'algorithme de descente de gradient par lots. En descente de gradient par lots, chacun l'itration effectue la mise jour

$$ \theta_j = \theta_j - \alpha \frac{1}{m} \sum_{i=1}^m \left( h_\theta(x^{(i)}) - y^{(i)}\right)x_j^{(i)} \qquad \text{simultaneously update } \theta_j \text{ for all } j$$

A chaque tape de descente de gradient, vos paramtres $ \theta_j $ se rapprochent des valeurs optimales qui permettront d'atteindre le cot le plus bas J ($ \theta $).

** Note d'implmentation: ** Nous stockons chaque exemple sous forme de ligne dans la matrice $ X $ en Python `numpy`. Pour prendre en compte le terme d'interception ($ \theta_0 $), nous ajoutons une premire colonne supplmentaire $ X $ et la dfinissons sur toutes les autres. Cela nous permet de traiter $ \theta_0 $ comme simplement une autre fonctionnalit.

    """

        ajout aux fonctionnalits, nous avons donc n + 1 colonnes.
    
    y: array_like
        Les valeurs de la fonction  chaque point de donnes. Ceci est un vecteur de
        forme (m,).
    
    thta: array_like

        forme (n + 1,).
    
    Retour
    -------

        La valeur de la fonction de cot de rgression.
    
    Instructions
    ------------
    Calculez le cot d'un choix particulier de thta.
    Vous devez dfinir J au cot.
    """
    
    # initialize some useful values
    m = len(y) #longueur de y

    # You need to return the following variables correctly

Vecteurs et matrices en numpy - Notes importantes de mise en uvre

Un vecteur dans numpy est un tableau unidimensionnel, par exemple np.array ([1, 2, 3]) est un vecteur. Une matrice dans numpy est un tableau deux dimensions, par exemplenp.array ([[1, 2, 3], [4, 5, 6]]). Cependant, ce qui suit est toujours considr comme une matrice np.array ([[1, 2, 3]]) car il a deux dimensions, mme s'il a une forme de 1x3 (qui ressemble un vecteur).

Compte tenu de ce qui prcde, la fonction np.dot que nous utiliserons pour toute multiplication matrice / vecteur a les proprits suivantes:

  • Il effectue toujours des produits intrieurs sur des vecteurs. Si x = np.array ([1, 2, 3]), alors np.dot (x, x) est un scalaire.
  • Pour la multiplication matrice-vecteur, donc si $ X $ est une matrice $ m \times n $ et $ y $ est un vecteur de longueur $ m $, alors l'opration np.dot (y, X) considre $ y $ comme vecteur $ 1 \times m $. En revanche, si $ y $ est un vecteur de longueur $ n $, alors l'opration np.dot (X, y) considre $ y $ comme un vecteur $ n \times 1 $.
  • Un vecteur peut tre promu en matrice en utilisant y [None] ou [y [np.newaxis]. Autrement dit, si y = np.array ([1, 2, 3]) est un vecteur de taille 3, alors y [None,:] est une matrice de forme $ 1 \times 3 $. Nous pouvons utiliser y [:, None] pour obtenir une forme de $ 3 \times 1 $.

def gradientDescent(X, y, theta, alpha, num_iters):
    """
    
    Effectue une descente de gradient pour apprendre `theta`. Met  jour le thta en prenant `num_iters`
    pas de gradient avec taux d'apprentissage alpha.
    
    Paramtres
    ----------
    X: array_like
        Jeu de donnes d'entre de forme (m x n + 1).
    
    y: arra_like
        Valeur aux caractristiques donnes. Un vecteur de forme (m,).
    
    thta: array_like
        Valeurs initiales des paramtres de rgression linaire.
        Un vecteur de forme (n + 1,).
    
    alpha: float
        Le taux d'apprentissage.
    
    num_iters: int
        Le nombre d'itrations pour la descente de gradient.
    
    Retour
    -------
    thta: array_like
        Les paramtres de rgression linaire appris. Un vecteur de forme (n + 1,).
    
    J_history: liste
        Une liste python pour les valeurs de la fonction de cot aprs chaque itration.
    
    Instructions
    ------------
    Effectuer une seule tape de gradient sur le vecteur de paramtres thta.

    Lors du dbogage, il peut tre utile d'imprimer les valeurs de
    la fonction de cot (computeCost) et le gradient ici.
    """
    # Initialize some useful values
    
    
    # make a copy of theta, to avoid changing the original array, since numpy arrays
    # are passed by reference to functions
    theta = theta.copy()
    m = len(y) #le nombre d'lments dans notre vecteur y (ie le nombre de lignes de notre dataset)
    J_history = [] # Use a python list to save cost in every iteration
    
    
    # ==================== YOUR CODE HERE =================================
    for i in range(num_iters):
            J = computeCost(X, y, theta)
            
            G = (X.dot(theta) - y) #diffrence entre le produit (matrice X et du vecteur theta) et du vecteur y
            theta = theta - alpha * (1/m * X.T.dot(G)) #gradient descent
            J_history.append(J)
            
        
    # =====================================================================
        
    # save the cost J in every iteration
    
    
    return theta, J_history

def  featureNormalize(X):
    """
    
    Normalise les fonctionnalits de X. renvoie une version normalise de X o
    la valeur moyenne de chaque caractristique est 0 et l'cart-type
    est 1. Ceci est souvent une bonne tape de prtraitement  faire lorsque vous travaillez avec
    algorithmes d'apprentissage.
    
    Paramtres
    ----------
    X: array_like
        L'ensemble de donnes de forme (m x n).
    
    Retour
    -------
    X_norm: array_like
        L'ensemble de donnes normalis de forme (m x n).
    
    Instructions
    ------------
    Tout d'abord, pour chaque dimension d'entit, calculez la moyenne de l'entit
    et le soustraire de l'ensemble de donnes, en stockant la valeur moyenne en mu.
    Ensuite, calculez l'cart type de chaque entit et divisez
    chaque caractristique par son cart-type, stockant l'cart-type
    dans sigma.
    
    Notez que X est une matrice o chaque colonne est une entit et chaque ligne est
    un exemple. Vous devez effectuer la normalisation sparment pour chaque fonction.
    
    Allusion
    ----
    Vous pourriez trouver les fonctions np.mean et np.std utiles.
    """
    # You need to set these values correctly
    X_norm = X.copy() 
    mu = np.zeros(X.shape[1])
    sigma = np.zeros(X.shape[1])
    n = X_norm.shape[1]
    # =========================== YOUR CODE HERE =====================
    i = 0
    for col in range(X.shape[1]):
        mu[i] = np.mean(X[:,col])
        sigma[i] = np.std(X[:,col])
        i += 1
        
    for row in range(X.shape[0]):
        for col in range(X.shape[1]):
            X_norm[row, col] = (X[row, col]) - mu[col]/sigma[col]
    
    # ================================================================
    return X_norm, mu, sigma

def gradientDescentMulti(X, y, theta, alpha, num_iters):
    """
    Performs gradient descent to learn theta.
    Updates theta by taking num_iters gradient steps with learning rate alpha.
        
    Parameters
    ----------
    X : array_like
        The dataset of shape (m x n+1).
    
    y : array_like
        A vector of shape (m, ) for the values at a given data point.
    
    theta : array_like
        The linear regression parameters. A vector of shape (n+1, )
    
    alpha : float
        The learning rate for gradient descent. 
    
    num_iters : int
        The number of iterations to run gradient descent. 
    
    Returns
    -------
    theta : array_like
        The learned linear regression parameters. A vector of shape (n+1, ).
    
    J_history : list
        A python list for the values of the cost function after each iteration.
    
    Instructions
    ------------
    Peform a single gradient step on the parameter vector theta.

    While debugging, it can be useful to print out the values of 
    the cost function (computeCost) and gradient here.
    """
    # Initialize some useful values
    m = y.shape[0] # number of training examples
    
    # make a copy of theta, which will be updated by gradient descent
    theta = theta.copy()
    
    J_history = []
    for i in range(num_iters):
        J = computeCostMulti(X, y, theta)
        G = X.dot(theta) - y
        theta = theta - ((alpha/m) * np.dot(G, X))
        J_history.append(J)
        
    
    return theta, J_history

def normalEqn(X, y):
    """
    Computes the closed-form solution to linear regression using the normal equations.
    
    Parameters
    ----------
    X : array_like
        The dataset of shape (m x n+1).
    
    y : array_like
        The value at each data point. A vector of shape (m, ).
    
    Returns
    -------
    theta : array_like
        Estimated linear regression parameters. A vector of shape (n+1, ).

    Complete the code to compute the closed form solution to linear
    regression and put the result in theta.

    Look up the function `np.linalg.pinv` for computing matrix inverse.

    theta = np.zeros(X.shape[1])

    G1 = 1/(np.dot(X.T,X))
    G2 = np.dot(X.T, y)

    return theta
